# Design

## 설계 원칙

### 1. 플랫폼 특화 콘텐츠
각 플랫폼(블로그/인스타/유튜브)의 알고리즘과 사용자 행동 패턴에 맞춘 콘텐츠를 생성한다. 하나의 피부관리 주제를 세 가지 다른 형식으로 변환하되, 각각의 플랫폼에 최적화한다.

### 2. 멀티 모델 활용
단일 AI 모델에 의존하지 않고 용도별로 최적의 모델을 사용한다:
- Claude: 구조적 분석, 전략 수립, 긴 글 작성
- GPT: 감성적 카피, 짧은 텍스트
- Gemini: 이미지 생성
- Kling AI: 영상 생성

### 3. 검수 우선 워크플로우
자동 게시가 아닌 "생성 → 검수 → 수동 업로드" 워크플로우를 따른다. 피부관리실 운영자가 콘텐츠를 확인하고 필요시 수정한 후 직접 올릴 수 있다.

### 4. 설정 기반 개인화
`config/shop_profile.yaml`에 관리실 정보를 상세히 기록하여 모든 콘텐츠가 해당 관리실의 브랜드 톤에 맞게 생성된다.

### 5. 의료법 준수
피부관리실은 의료기관이 아니므로, 의료 행위를 암시하는 표현(진단, 치료, 처방 등)을 사용하지 않도록 프롬프트에서 제어한다.
